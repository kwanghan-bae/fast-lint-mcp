# 변경 이력 (Changelog)

## [2.1.0] - 2026-02-26 (목요일)

### 추가 ✨
- **Zero-Cache 실시간 분석 아키텍처**: SQLite 기반 캐시 시스템을 제거하고 매 실행 시 실시간으로 전체 분석을 수행하여 규칙 업데이트의 즉각적인 반영을 보장합니다.
- **범용 주석 강제 (ESM & CJS)**: ESM(`export`) 뿐만 아니라 Node.js의 **CommonJS(`module.exports`, `exports`)** 패턴까지 인식하여 한글 주석 작성을 강제합니다.
- **클래스 멤버 전수 조사**: 메서드, 멤버 변수(`Field`), 인터페이스, 타입 별칭에 대한 가독성 검사 로직을 추가했습니다.
- **지능형 주석 탐색**: 코드와 주석 사이에 빈 줄이 있어도 위로 **최대 5줄**까지 추적하여 주석을 찾아내는 로직을 도입했습니다.
- **리포트 통계 강화**: 리포트 하단에 **서버 버전(v2.1.0) 및 실제 분석된 파일 개수**를 명시하여 분석의 투명성을 높였습니다.
- **자바스크립트 전용 검증**: `.js` 환경에서의 AST 차이를 극복하기 위한 전용 테스트 케이스(`tests/analysis_reviewer_js.test.ts`)를 추가했습니다.

### 변경 ⚡
- **Pure Native Rust 전환 완료**: 무거운 `ts-morph` 의존성을 100% 제거하고 Rust 기반 AST 엔진(`ast-grep`)으로 시스템을 단일화했습니다.
- **경로 하드코딩 제거**: `src/` 디렉토리 제약을 없애고 프로젝트 루트부터 모든 파일을 분석 대상으로 포함합니다.
- **타입 안정성 강화**: 프로젝트 전반의 `any` 타입을 제거하고 명확한 인터페이스를 적용하여 런타임 안정성을 확보했습니다.
- **Markdown 기반 품질 리포트**: MCP 클라이언트에서 최적의 가독성을 제공하기 위해 Markdown 테이블과 이모지 기반의 시각적 리포트로 교체했습니다.

## [2.0.0] - 2026-02-25 (수요일)

### 추가 ✨
- **네이티브 Rust 의존성 엔진**: 무거운 `ts-morph`와 실험적인 정규식 기반 추적을 대체하여 **Native Rust (@ast-grep/napi)** 기반의 초고속 엔진을 도입했습니다.
- **진짜 전역 미사용 코드 탐지 (Find Dead Code)**: 프로젝트 내 모든 Export 심볼에 대해 실제로 사용되지 않는 코드를 정밀하게 찾아냅니다.
- **아키텍처 가드레일 (Architecture Guardrail)**: 레이어 간 의존성 방향을 강제하는 시스템을 도입했습니다.
- **자가 치유 워크플로우 (Self-Healing Workflow)**: 에이전트가 코드를 수정하고 스스로 검증하는 자율 루프를 구축했습니다.
