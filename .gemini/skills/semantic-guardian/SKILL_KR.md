# SKILL: 세만틱 가디언 (Fast-Lint v2.0) - 한글 해설서

이 파일은 AI 에이전트가 사용하는 영문 `SKILL.md`의 내용을 사용자가 읽기 쉽게 번역한 문서입니다.

## 📋 스킬 개요
TypeScript/JavaScript 프로젝트의 코드 품질을 심볼(함수, 클래스) 단위로 정밀하게 분석하고 관리하는 전문 지식을 제공합니다. 
단순한 텍스트 분석을 넘어, LSP 수준의 정밀한 코드 추적과 최소 토큰 전략을 수행하는 것이 핵심입니다.

## 🧠 전문가 워크플로우 (SOP)

### 워크플로우 1: 정밀 탐색 및 수정 (Precision Navigation & Edit)
**목표:** 파일 전체를 읽지 않고 필요한 부분만 수정하여 토큰(비용)을 획기적으로 절약합니다.
1. **문제 식별**: `quality-check`로 수정이 필요한 파일을 찾습니다.
2. **구조 분석**: `get-symbol-metrics`를 호출하여 해당 파일 내의 함수/클래스 목록과 각 심볼의 복잡도를 확인합니다.
3. **정밀 읽기**: 수정할 특정 함수만 `get-symbol-content`로 읽어 들입니다. **절대로 대용량 파일을 전체 읽기(`read_file`) 하지 마십시오.**
4. **영향도 평가**: 수정 전에 `analyze-impact`를 호출하여 이 수정이 다른 파일이나 테스트에 어떤 영향을 줄지 미리 확인합니다.

### 워크플로우 2: 기술 부채 및 구조 최적화 (Debt & Structure)
**목표:** 프로젝트의 복잡도를 낮추고 깨끗한 코드 베이스를 유지합니다.
1. **미사용 코드 탐지**: `find-dead-code`로 프로젝트 전체에서 호출되지 않는 수출(export) 심볼을 찾아 삭제를 제안합니다.
2. **구조적 점검**: 모듈 간의 순환 참조나 복잡한 임포트 그래프를 감시하여 리팩토링 포인트를 제안합니다.

## 🛡️ 규칙 및 제약 사항
- **최소 토큰 원칙**: 항상 파일 단위보다는 심볼(함수) 단위의 조작을 우선시하십시오.
- **안전 우선**: 공통 심볼을 수정하기 전에는 반드시 `analyze-impact` 결과를 보고하십시오.
- **데이터 기반 보고**: 문제를 보고할 때 "복잡함"과 같은 모호한 표현 대신 "복잡도: 25, 라인: 45~80"과 같은 구체적인 수치를 사용하십시오.
