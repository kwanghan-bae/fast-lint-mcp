# SKILL: 세만틱 가디언 (Fast-Lint v2.0) - 한글 해설서

이 파일은 AI 에이전트가 사용하는 영문 `SKILL.md`의 내용을 사용자가 읽기 쉽게 번역한 문서입니다.

## 📋 스킬 개요
TypeScript/JavaScript 프로젝트의 코드 품질을 심볼(함수, 클래스) 단위로 정밀하게 분석하고 관리하는 전문 지식을 제공합니다. 
단순한 텍스트 분석을 넘어, 자가 치유(Self-Healing) 루프와 고정밀 코드 네비게이션을 통해 최고의 자율성과 토큰 효율성을 달성하는 것이 핵심입니다.

## 🧠 전문가 워크플로우 (SOP)

### 워크플로우 1: 고정밀 탐색 및 수정 (High-Precision Navigation & Edit)
**목표:** 컨텍스트 낭비 없이 복잡한 코드베이스를 탐색하고 수정합니다.
1. **구조적 매핑**: `find-references`와 `go-to-definition`을 사용하여 코드 간의 관계를 매핑합니다.
2. **정밀 분석**: `get-symbol-metrics`를 호출하여 수정 목표를 정확히 식별합니다.
3. **타겟 읽기**: 필요한 특정 코드 조각만 `get-symbol-content`로 읽어 들입니다. **대용량 파일을 전체 읽기(`read_file`) 하지 마십시오.**
4. **영향도 평가**: 수정 전에 `analyze-impact`를 호출하여 사이드 이펙트를 미리 파악합니다.

### 워크플로우 2: 자율형 자가 치유 루프 (Autonomous Self-Healing Loop)
**목표:** 모든 변경 사항이 최종 확정되기 전에 기능적으로 검증되었음을 보장합니다.
1. **변경 적용**: 코드를 수정합니다.
2. **실행 검증**: `verify-fix`를 적절한 테스트 명령어(예: `npm test`)와 함께 실행합니다.
3. **반복 수리**: 만약 테스트가 실패하면, 에러 로그를 분석하고 코드를 조정한 뒤 성공할 때까지 반복합니다.

### 워크플로우 3: 아키텍처 무결성 유지 (Architectural Integrity)
**목표:** 프로젝트 전체의 설계 규칙을 강제합니다.
1. **규칙 검증**: `quality-check` 결과에서 `ARCHITECTURE_VIOLATION`이 발생하는지 감시합니다.
2. **기술 부채 정리**: `find-dead-code`를 사용하여 사용되지 않는 수출(export) 심볼을 정리합니다.

## 🛡️ 규칙 및 제약 사항
- **자율 검증 원칙**: `verify-fix`가 통과하기 전까지는 작업을 "완료"된 것으로 간주하지 마십시오.
- **최소 토큰 원칙**: 항상 파일 단위보다는 심볼(함수) 단위의 조작을 우선시하십시오.
- **데이터 기반 추론**: 리팩토링 제안 시 구체적인 메트릭과 의존성 맵을 근거로 사용하십시오.
